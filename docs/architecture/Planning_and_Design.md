# 集团研发主数据系统 (MDM) 建设规划方案 v4.0

## 1. 核心诉求与解决痛点
当前研发数据存在“孤岛效应”，GitLab、Jira、SonarQube 账号体系不一，导致无法从集团视角审计人力投入与产出价值。本方案旨在构建以“人、项、物”为核心的研发黄金数据源。

## 2. 演进路线图 (Phase-based Roadmap)

### 第一阶段：底座升维 (Month 1-2)
* **目标**：实现数据从“采集”到“治理”的转型。
* **关键任务**：
    * 引入 **dbt** 替换现有的原生 SQL View，解决逻辑黑盒问题。
    * 建立 **Identity Resolution (身份归一)** 引擎，消除跨工具账号差异。

### 第二阶段：价值穿透 (Month 3-5)
* **目标**：打通财务维度，落地 FinOps。
* **关键任务**：
    * 对接 HR/财务系统，实现研发工时到金额的自动折算。
    * 落地 **DORA 指标体系**，建立部门效能红黑榜。

### 第三阶段：智能与合规 (Month 6+)
* **目标**：构建知识图谱与自动审计。
* **关键任务**：
    * 构建“技术栈专家图谱”，实现人才精准盘点。
    * 自动化审计证据链生成，支持高新企业认定。

## 3. 预期成效
* **数据准确率**：身份关联准确率提升至 99% 以上。
* **报表效率**：集团级效能报表生成从 3 天缩短至秒级。
